# Pi 借鉴改进实施规格（轻量版）

> 状态：Draft（仅完成规格与测试设计，尚未实现功能）  
> 来源：[PI_借鉴与改进建议.md](./PI_借鉴与改进建议.md)  
> 目标：用最少的代码补齐安全边界、方案试错和结果复盘能力，适合日常实习项目展示。

## 1. 规格流程

每项改进按以下顺序推进：

1. 先确认问题、范围和验收标准。
2. 先写失败测试，再实现最小功能。
3. 运行单元测试、集成测试和必要的手工演示。
4. 测试通过后才在 README、简历或面试材料中写成“已实现”。

本规格只规划三个增量功能，不改变现有 LLM、MCP、权限和上下文压缩机制。

## 2. 目标与非目标

### 目标

- 陌生仓库的项目级配置在启动阶段默认不获得信任，避免 Hook、命令、技能或 MCP 子进程悄悄影响本机。
- 用户可以从当前会话复制出独立分支，比较方案而不污染原会话。
- 用户可以导出脱敏后的会话记录，作为调试和面试展示材料。
- 保持旧配置、旧 JSONL 会话和现有命令的兼容性；失败时给出可理解的提示。

### 非目标

- 不实现 Pi 的完整会话树、TypeScript 扩展生态或插件安装器。
- 不增加新的模型厂商、账号体系、桌面端或容器运行时。
- 不在本阶段实现非交互式 CLI 的完整自动授权流程。
- 不以“默认绕过确认”换取演示速度。

## 3. 范围与优先级

| ID | 功能 | 优先级 | 预估 | 主要代码边界 |
| --- | --- | --- | --- | --- |
| P0 | 项目本地配置“信任门槛” | 最高 | 0.5～1 天 | `internal/config/`、`internal/hooks/`、`internal/commands/`、`internal/skills/`、TUI |
| P1 | `/fork` 会话分支 | 中 | 1～2 天 | `internal/session/`、`internal/tui/` |
| P2 | `/export` 脱敏 HTML 导出 | 低 | 0.5～1 天 | `internal/session/`、`internal/tui/` |

实施顺序固定为 P0 → P1 → P2；任何阶段未达到验收标准，都不进入下一阶段。

## 4. 功能规格

### 4.1 P0：项目本地配置信任门槛

#### 行为要求

- 启动时先识别项目根目录及项目级 `.mygocode/` 资源；首次发现且没有决定时，TUI 只询问一次“是否信任此项目的本地配置”。
- 信任记录保存在用户目录 `~/.mygocode/trusted-projects.json`，键使用项目根目录的规范化绝对路径；文件权限按用户私有文件处理。
- 选择“信任”后，项目级配置、Hook、commands、skills 和 MCP 配置按现有流程加载。
- 选择“不信任”或信任记录不存在时，只启用全局配置；不得执行项目 Hook，不注册项目 commands/skills，不启动项目配置声明的 MCP 子进程。
- 信任文件不存在、无法解析或项目路径发生变化时，回到“未决定”状态，不能静默信任。
- `/trust` 至少支持查看当前项目状态和撤销当前项目的信任；输出不得包含 API key、Header 或完整配置原文。
- 远程 UI/未来 CLI 尚未提供确认交互时，采用“未信任”默认值并提示用户回到 TUI 确认。

#### 验收标准

- 未信任项目启动后，带 `pre_tool_use` 的项目 Hook 不会产生副作用。
- 信任项目启动后，现有项目 Hook、命令和技能行为与改动前一致。
- 同一个规范化项目路径重复启动不再重复询问；撤销后下一次启动会再次询问。
- 信任存储损坏不会导致启动崩溃，也不会导致自动放行。

#### 建议测试

- 新增信任存储包的单元测试：首次状态、写入/读取、撤销、损坏 JSON、路径规范化。
- 配置加载集成测试：未信任时跳过项目 Hook/commands/skills/MCP；信任后完整加载。
- 回归 `internal/hooks/hooks_test.go` 和 `internal/commands/loader_test.go`，确认全局资源仍然可用。

### 4.2 P1：`/fork` 会话分支

#### 行为要求

- `/fork` 以当前会话为源，创建新的会话 ID；源 JSONL 保持不变，新会话从源会话当前状态开始。
- 新会话首条元数据记录保存 `parent_session_id`、`forked_at` 和版本字段；元数据不应被当作普通对话消息发送给模型。
- 复制过程使用临时文件后原子改名；源会话不存在、读取失败或目标写入失败时，不留下半成品会话。
- `/session info` 显示当前 ID、父会话 ID、消息数和文件大小；无父会话的旧会话继续正常显示。
- `/resume` 列表对分支会话增加简短的“from <parent>”标记，不改变原有搜索和排序。

#### 验收标准

- 分支会话可以独立追加消息；追加内容不会出现在父会话中。
- 包含 `compact_boundary` 的会话分支后，恢复结果与源会话当前压缩状态一致。
- 旧格式 JSONL（没有元数据）仍可恢复，且不会被迁移或重写。
- 失败场景不会产生无法恢复的空文件或错误父子关系。

#### 建议测试

- `internal/session/session_test.go`：完整复制、父子元数据、源文件不变、空会话、损坏/不存在源文件、压缩边界兼容。
- TUI 命令测试：`/fork` 返回新 ID，`/session info` 和 `/resume` 正确显示父子关系。
- 回归 `go test ./internal/session ./internal/tui`。

### 4.3 P2：`/export` 脱敏 HTML 导出

#### 行为要求

- `/export` 默认写入 `.mygocode/exports/session-<id>.html`；可选路径仅允许项目目录内的相对文件名，拒绝绝对路径和 `..` 穿越。
- 导出只读当前会话 JSONL，不改变会话内容、压缩边界或恢复结果。
- HTML 展示会话 ID、时间、消息角色、消息内容和已有的记录类型；工具结果按固定上限截断。
- 所有消息文本经过 HTML 转义；明显的 API key、Bearer token、密码和配置密钥值在导出中必须被替换为 `[REDACTED]`。
- 不读取或嵌入 `config.yaml`、环境变量或其他项目文件；导出失败时保留原 JSONL。

#### 验收标准

- 导出的 HTML 可以在无网络环境下打开，且包含用户问题和助手回复。
- 输入中出现 `<script>`、`&` 等字符时不会形成可执行 HTML。
- 工具结果超过上限时稳定截断，不因单条大消息耗尽内存。
- 导出前后 `LoadSession` 返回的消息完全一致。

#### 建议测试

- `internal/session/export_test.go`：HTML 转义、脱敏、截断、默认路径、非法路径、空/不存在会话。
- 导出集成测试：写入 HTML 后重新读取 JSONL，断言消息数、内容和压缩边界不变。
- 手工检查：浏览器离线打开导出文件，确认没有外链、密钥或完整配置。

## 7. 风险与取舍

- 信任门槛会增加首次进入项目的一次交互，但这是安全边界，不应通过默认信任消除。
- 会话分支先复制整份 JSONL，不做内容去重或树形 UI；以实现简单和可恢复为优先。
- HTML 导出只做脱敏和固定长度限制，不承诺完整审计或安全取证能力。
- 若实现过程中发现现有结构需要大范围重构，应回到规格重新拆分，不把 P0～P2 扩展成新的平台工程。
